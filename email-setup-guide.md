# Self-Hosted Mail Server Setup Guide (Stalwart + Coolify)

This guide documents the exact configuration and steps used to set up the professional email server (`info@tradekav.ir`) using **Stalwart Mail Server** deployed inside **Coolify** on your VPS.

---

## 1. Coolify Configuration (Docker Compose)

Deploy Stalwart in Coolify as a **Docker Compose** service stack.

### Compose File (`docker-compose.yml`)
Use the following YAML configuration. It exposes all required mail protocols, maps the web UI to port `8080` internally, and maps data directories directly to the host system for persistence.

```yaml
version: '3.8'

services:
  stalwart-mail:
    image: ghcr.io/stalwartlabs/stalwart:latest
    container_name: stalwart-mail
    restart: always
    ports:
      # SMTP ports
      - "25:25"     # SMTP (Incoming mail - requires Port 25 open on VPS provider)
      - "465:465"   # SMTPS (Implicit TLS for sending mail)
      - "587:587"   # SMTP Submission (Explicit TLS / STARTTLS for sending mail)
      
      # IMAP ports
      - "993:993"   # IMAPS (Secure IMAP for reading mail)
      - "143:143"   # IMAP (STARTTLS)
      
      # POP3 ports
      - "995:995"   # POP3S (Secure POP3 for fetching mail - needed for Gmail import)
      - "110:110"   # POP3
    volumes:
      # Mount configuration and data directly to the VPS host for permanent storage
      - /data/coolify/source/stalwart/etc:/etc/stalwart
      - stalwart-data:/var/lib/stalwart
    environment:
      - TZ=Asia/Tehran
    networks:
      - default
      - coolify

volumes:
  stalwart-data:
    driver: local

networks:
  coolify:
    name: coolify
    external: true
```

### Domain Routing Settings in Coolify GUI
To route `https://mail.tradekav.ir` through Coolify's Traefik proxy to Stalwart's web panel:
1. Open the settings of the `stalwart-mail` child container in Coolify.
2. Under **Domains**, set the value to:
   ```text
   https://mail.tradekav.ir:8080
   ```
   *(Appending `:8080` tells Traefik to route the secure web traffic to the internal port 8080 without binding port 8080 to the public host IP).*

## 2. Restoring Existing Configurations (If moving volumes)
If you need to restore or move configurations to the host folder `/data/coolify/source/stalwart/etc/` from an old named volume:
```bash
# 1. Copy files from old named volume to host directory
sudo cp -r /var/lib/docker/volumes/<OLD_VOLUME_NAME>/_data/* /data/coolify/source/stalwart/etc/

# 2. Fix permissions so the container can read/write
sudo chown -R 1000:1000 /data/coolify/source/stalwart/etc/
```

## 3. SSL/TLS Certificate Setup (Let's Encrypt)

Because mail protocols require a valid, trusted SSL/TLS certificate, we extract the certificate generated by Coolify's Traefik proxy and share it with Stalwart.

### Step A: Run the Extraction Script
Run this Python script on the VPS host. It decodes the certificate for `mail.tradekav.ir` from Traefik's `acme.json` and writes it to Stalwart's config directory:

```bash
sudo python3 -c "
import json, base64, os
try:
    with open('/data/coolify/proxy/acme.json') as f:
        data = json.load(f)
    
    certificates = []
    for resolver in data.values():
        if isinstance(resolver, dict) and 'Certificates' in resolver:
            certificates.extend(resolver['Certificates'])
            
    found = False
    for cert in certificates:
        domain = cert.get('domain', {}).get('main', '')
        sans = cert.get('domain', {}).get('sans', [])
        if domain == 'mail.tradekav.ir' or 'mail.tradekav.ir' in sans:
            os.makedirs('/data/coolify/source/stalwart/etc', exist_ok=True)
            
            # Save Certificate
            with open('/data/coolify/source/stalwart/etc/mail.crt', 'wb') as out:
                out.write(base64.b64decode(cert['certificate']))
            
            # Save Private Key
            with open('/data/coolify/source/stalwart/etc/mail.key', 'wb') as out:
                out.write(base64.b64decode(cert['key']))
                
            print('Successfully extracted SSL certificate and key for mail.tradekav.ir!')
            found = True
            break
            
    if not found:
        print('Error: Could not find certificate for mail.tradekav.ir inside acme.json')
except Exception as e:
    print('Failed to extract certificate:', str(e))
"
```

### Step B: Import into Stalwart Web UI
1. Log into your Stalwart Admin Panel: `https://mail.tradekav.ir`.
2. Go to **Settings** -> **Security** -> **Certificates**.
3. Click the **`+` (Add)** button and select **Import Certificate** (PEM/Files).
4. Enter the paths inside the container:
   * **Private Key Path:** `/etc/stalwart/mail.key`
   * **Certificate Path:** `/etc/stalwart/mail.crt`
5. Click **Save**. The certificate will now apply to SMTP and IMAP listeners.

## 4. DNS Configuration

Add the following DNS records to your DNS provider (e.g. Afranet) for email deliverability:

| Record Type | Name (Host) | Value/Target | Priority / Notes |
| :--- | :--- | :--- | :--- |
| **A** | `mail` | `<YOUR_VPS_PUBLIC_IP>` | Points mail subdomain to VPS |
| **MX** | `@` | `mail.tradekav.ir` | Priority: `10` (Receives email) |
| **TXT (SPF)** | `@` | `v=spf1 ip4:<YOUR_VPS_PUBLIC_IP> -all` | Authorizes VPS to send emails |
| **TXT (DMARC)**| `_dmarc` | `v=DMARC1; p=quarantine; pct=100; rua=mailto:info@tradekav.ir` | Security policy against spoofing |
| **TXT (DKIM RSA)**| `v1-rsa-20260629._domainkey` | `v=DKIM1; k=rsa; p=<RSA_PUBLIC_KEY>` | Cryptographic signatures (Get value from Stalwart Domains panel) |
| **TXT (DKIM Ed)**| `v1-ed25519-20260629._domainkey`| `v=DKIM1; k=ed25519; p=<ED25519_PUBLIC_KEY>` | Cryptographic signatures (Get value from Stalwart Domains panel) |

## 5. Connecting Mail Clients

### IMAP & SMTP Settings (Outlook, Thunderbird, etc.)
* **Username:** `info@tradekav.ir`
* **Password:** *(The password you created for the user `info` in Stalwart Users dashboard)*
* **Incoming Server (IMAP):** `mail.tradekav.ir`
  * **Port:** `993`
  * **Encryption:** `SSL/TLS`
* **Outgoing Server (SMTP):** `mail.tradekav.ir`
  * **Port:** `465` (or `587` with `STARTTLS`)
  * **Encryption:** `SSL/TLS`

### Gmail Integration (Send Mail As)
To send emails as `info@tradekav.ir` directly inside your personal Gmail account:
1. In Gmail, go to **Settings** -> **Accounts and Import** -> **Send mail as** -> **Add another email address**.
2. Name: `tradekav`, Email: `info@tradekav.ir`.
3. **SMTP Server:** `mail.tradekav.ir`
4. **Port:** `465` (SSL)
5. **Username:** `info@tradekav.ir`
6. **Password:** your inbox password.
7. Click **Add Account** and complete the verification using the email code sent to your Outlook client.
